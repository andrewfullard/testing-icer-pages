    Scratch File Systems

    In addition to home and research files, we have file systems for temporary work called "scratch" available to users.  Right now, HPCC has two scratch file systems. One (gs18) is installed in November 2018 and based on the IBM General Parallel File System (GPFS). It is fast and reliable located at /mnt/gs18 with a capacity of 834 TB. The other (ls15) is installed in 2015 and based on Lustre file system (LFS). It is located at /mnt/ls15 with a capacity of 1.9 PB.  Both of them are housed in the new MSU data center and have a high speed network (infiniband) connection.

    Note

    A new scratch system (gs21) is install on June 2022 and will replace the old scratch system (gs18 and ls15). By July 31 2022, old scratch space will be retuired and not accessible. Users are requested to migrate their files from old scratch system to new scratch system before ls15 and gs18 become unavailable.

    Unlike the home and research directories, the scratch spaces are not intended for long-term storage and can not be accessed from the gateway node (except rsync gateway). Files saved in the systems have no back-up. Users should use it as a working space for temporary data file storage and save their results or data back to HOME or RESEARCH space after jobs finish running. Any file not modified in 45 days may be deleted to keep the system stable and available to all users. The quota of the space for each user is limited to total 50TB and less than 1 Million files. Research groups can also have their scratch folders available. If you would like a folder for your research group please request one.  

    Note

    New scratch system gs21 has the same quota and purge policies enforced as on gs18 and ls15.

    HPCC users can find their scratch spaces from the two paths: 

    ls15 scratch : /mnt/ls15/scratch/users/$USER  (or use variable$SC15)

    gs18 scratch : /mnt/gs18/scratch/users/$USER(or use variable$SC18or$SCRATCH) or the shortcut  /mnt/scratch/$USER

    where $USER is the system variable of your HPCC user id. Users can run "quota" command to check the usage limit of their two spaces.

    Note

    New gs21 scratch space can be found at /mnt/gs21/scratch/$USER(or use variable $SCRATCH or the shortcut  /mnt/scratch/$USER after old scratch system retired).

    To find out files in the scratch space almost getting purged, users can run the following commands:

    1
2
    find $SC18 -type f -mtime +40
find $SC15 -type f -mtime +40

    It will list files not modified more than 40 days. Users may set different days in the command to check their files.

    Notice on Using Scratch Systems

    The parallel file systems are designed for heavy reading and writing (I/O) of large files. Both of them are highly scalable and can sustain very high data transfer rates. It is a good choice for running parallel programs on multiple nodes with I/O access to many files. For many types of jobs, it's much faster than using your home or research folders. However, for ls15 system, it is not good at handling high frequency I/O operations on a lot of small files. For those type of jobs you can use gs18 or explore using local file system at $TMPDIR (/mnt/local) . Please contact us for more information if you find the Scratch file system is too slow for your work.

    Because the scratch space is intended for job running, it cannot be accessed from the general gateway nodes. That is, after you log in to HPCC (ssh to hpcc.msu.edu) and stay on the gateway. If you try to access scratch, you will receive an error  "No such file or directory."   To access scratch, please connect to a development node (e.g. ssh dev-intel16).  Your scratch folder is also available to all of the compute nodes and hence your jobs. To transfer files to/from your scratch folder in hpcc, please use our file transfer gateway with the host name rsync.hpcc.msu.edu. See Transferring Files to the HPCC for more information.

    Copying Files between ls15 and gs18

    Users can copy files from ls15 to gs18 by 'cp' command:

    1
    $ cp -av /mnt/ls15/scratch/users/$USER /mnt/gs18/scratch/users/$USER

    or 'rsync' command:

    1
    $ rsync -av /mnt/ls15/scratch/users/$USER/ /mnt/gs18/scratch/users/$USER/

    rsync can be restarted and won't recopy files if you have to stop it or if it fails/interrupted during the copy.

    You can do this from any dev node. If you have many files this will take a long time so please plan to have your computer open during that time or use virtual terminal (such as screen). We are happy to help if you have any concerns or difficulty.  Please contact or visit us during office hours Mondays 1-2pm or Thursdays 1-2pm 1440 BPS building.

    Note

    The methods used to copy files bewteen ls15 and gs18 mentioned above could be used to migrate files from old scatch system (ls15 and gs18) to new scratch (gs21).

    Change Default SCRATCH

    If your scripts depend on the $SCRATCH variable to point to the ls15 scratch, as an immediate measure you can change it the value:

    1
    $ export SCRATCH=/mnt/ls15/scratch/users/$USER

    We don't recommend this. (We recommend copying your files to gs18 or use $SC15 instead.) However, if your transition to ls15 takes a long time, you could add this command to your scripts or when you log-in.

    Note

    After migrating files from old scratch systems to the new scratch system, users may want to check if they have used any variables in their script files to point to the old scratch system and make the proper update to point to the new scratch system.
